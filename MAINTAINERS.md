# Maintainers

This project is open to long-term contributors who want to help with review, triage, and release quality.

## What maintainers do

- review pull requests for correctness and regression risk
- help triage issues and route them to the right area
- keep docs and contribution paths clear
- protect the repo from leaking secrets or unsafe artifacts
- help keep parser and contract tests healthy

## What we look for

Potential maintainers usually:

- contribute multiple focused PRs
- write or improve tests alongside behavior changes
- communicate clearly in review
- show care around security and authenticated data
- improve the repo for the next contributor, not just for themselves

## Path to more responsibility

1. Start with small PRs or issue triage.
2. Build familiarity in one area such as Python contracts, Go CLI, frontend, or CI.
3. Review other contributors' changes constructively.
4. Help improve documentation and contributor experience.

If you are interested in helping regularly, open an issue or PR discussion and say which area you want to own.
